Tic Tac Toe

by: Praveena Ratnavel

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make Tic Tac Toe because it's a classic children's game that I thought could use a few twists, such as making different characters and colors.

2. What I Learned

From user testing, I learned that the audio sound was too long, and that it should be cut short. I also learned that if players click on boxes that already have images on them, the program changes these images, so I had to check that there was no image already in each box when the player clicks it. I got feedback from teachers and classmates to add in a system where people can play multiple games against the same person, so I added a reset option at the end of the game.

3. Architecture of My Game

The first class, the Image Choosing class, asks the different players for their names, introduces the game, and asks the players for which images they want to play with. The next class then takes these names and images and makes a Tic Tac Toe board, where players can click on which box they want. The last class is the rules, or logic, where the program checks if anyone won based on a series of if loops.

4. Technical Implementation

I had multiple JFrames with a Grid Layout with buttons. The first JFrame had buttons to decide which image was associated with the players. The second JFrame had buttons for players to click on to play the actual Tic Tac Toe game, and the players switch off turns to play. The aspect of the game that I am proud of is when players can decide which image they want to be. They can choose to be X's and O's, cats and dogs, violin and trumpet, or football and basketball. The first JFrame allows the players some freedom and they can chose which image they are. I first found the images, and then changed the size of them and then implemented them into the code depending on which image the player chose.

5. Most Challenging Part

The most challenging part of this project was to make all the if statements and rules, because it was in a 1D array, not a 2D array. I had to draw out the board and then assign each box an integer in an array, and then change that number based on if it has an image and which image it was. After making the if loops, one particular solution would not show that the player one because the if loops were true, but it was that there was no image in 3 separate boxes. I had to add another statement in each if loop, checking that there was an image in each box that it was checking.

6. Future Fixes and Updates

In the future, if I were to continue working on this game, I would add a user interface that keeps the names and number of games each person won in a neat display. The user would be able to change this display to their liking, and you would be able to add multiple players and then chose which player you are if you're coming back to play again.
